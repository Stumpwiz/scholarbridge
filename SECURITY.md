# Security Policy

## Supported Versions

Security fixes are applied to the latest main branch state.

## Reporting a Vulnerability

Please report suspected vulnerabilities privately to: `maintainer@example.org`.

Include:

- affected component or file
- reproduction steps or proof of concept
- impact assessment
- suggested mitigation (if available)

Please do not open public issues for unpatched vulnerabilities.

## Scope Notes

- This repository must not contain production credentials, production data exports, or real personal data.
- Security reports related to accidental sensitive-data commits are treated as high priority.
