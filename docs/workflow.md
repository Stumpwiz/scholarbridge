# Workflow Overview

This document captures known process flow and expected system evolution toward a lightweight donor solicitation management CRM. It should be updated as committee practices are clarified.

## Current Workflow (Spreadsheet-Centered)

1. Committee tracks partners and contacts in one or more Excel sheets.
2. Annual outreach targets are assembled manually for each solicitation season.
3. Contact attempts and outcomes are logged with varied conventions.
4. Donation outcomes are entered manually, often after email or paper confirmation.
5. Acknowledgment tasks are tracked through ad hoc notes and message threads.
6. Historical analysis is manual and dependent on continuity of spreadsheet structure.

## Anticipated Future Workflow (ScholarBridge-Enabled)

1. Import existing spreadsheet data into structured records.
2. Define a campaign for each annual solicitation period.
3. Create solicitations that associate partners and contacts to the active campaign.
4. Track outreach events, statuses, and donation commitments in one system.
5. Generate acknowledgment outputs from standardized data.
6. Review campaign outcomes using repeatable historical reporting views.

## Unknowns and Open Questions

- What is the committee's authoritative definition of a "patron" versus "donor"?
- Can one partner have multiple active contacts with different roles?
- What is the minimum required data for a valid solicitation?
- How should partial, pledged, or in-kind contributions be represented?
- What acknowledgment formats are required first (email, print letter, both)?
- Which existing spreadsheet columns are mandatory versus optional legacy artifacts?
- What audit expectations exist for edits, corrections, and year-over-year carryover?
